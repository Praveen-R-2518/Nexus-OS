## Notes

- `wf1/wf3/wf4_*.json` are **snapshots exported from the live instance** after the 2026-07-12
  tenant-stamping (2.4) + autopilot-rewire (1.5) edits. They are NOT generated by
  `build_n8n_workflow_exports.js` — if you edit these workflows live again, re-export the
  snapshot (via the n8n MCP `get_workflow_details` or the n8n UI export) to keep the repo in sync.
- All are `active: false`. Do not activate until the relevant preconditions are met (Gmail
  `gmail.send` scope for real sending; Supabase custom-auth credential wired on HTTP nodes).
- HTTP nodes that call Supabase REST use the instance's **Supabase custom auth** credential —
  re-select it on import. Nodes that call the Next.js app (`Autopilot Send (policy-gated)`) use
  header-based `Authorization: Bearer {{$env.N8N_INGEST_TOKEN}}` and need
  `NEXUS_APP_BASE_URL` + `N8N_INGEST_TOKEN` in the n8n environment.
